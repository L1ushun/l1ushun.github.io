首先来写一段 jsx 代码

Automatic Runtime

在新版本的 React 使用 Automatic Runtime 模式，使用 react/jsx-runtime 下的 jsx 和 jsxs 进行 jsx 转换。

Classic Runtime

旧版本使用 React.createElement 进行 jsx 转换，这就是为什么旧版本 jsx 文件需要引入 React 的原因。

jsx 注意事项

  1. JSX 必须只有一个根标签
  2. 布尔值、空值、undefined 不会显示。NaN 会显示
  3. {} 只能放 js 表达式，不能放语句（if、for）

组件

state

一次 setState 流程

触发setState -> 计算更新的优先级 -> 找到要更新的节点，合并 state -> render -> commit阶段,替换真实dom -> setState 的 callback

setState 同步还是异步

在 react 17 中，setState 是批量执行的，但如果在 异步环境中，如 setTimeout 等异步环境中，每次 setState 都会走一遍上面的流程，所以可以拿到更新后的值。
在 react 18 中，如果用 createRoot 的 api，就不会有这种问题了，所有更新都将自动批处理，所有的 setState 都是异步批量执行了。
